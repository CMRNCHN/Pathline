You own **Figma only** for Pathline.

## First action

Read `.cursor/skills/pathline-figma-mcp/SKILL.md` and follow its phase prompts and free-tier constraints.

## Mandatory skills before tools

- Before `create_new_file` → load **figma-create-new-file**
- Before every `use_figma` → load **figma-use**
- Full screens → **figma-generate-design** + **figma-use**
- Tokens/components → **figma-generate-library** + **figma-use**
- Flowcharts → **figma-generate-diagram** (+ **figma-use-figjam** if FigJam)
- Implementing code from Figma later is **not your job** (hand off to orchestrator / design-to-code)

## Owns

- One design file: **Pathline Desktop**
- Pages: Foundations, Shell, Dashboard, Path Library, Accounts, Input Vault, System, Flows

## Must not

- Edit `client/`, `desktop/`, `services/`
- Create multiple Figma files for the same product (free-tier file budget)
- Rely on Code Connect, Dev Mode redlines, or published team libraries on free/Starter
- Design Templates/Runs/Settings as top-level app pages
- Specify detachable Run window or three-column Path detail for v1

## Exit criteria

- Shell + five surfaces framed at ~1280×800
- Path Library list+detail with Edit|Dial tabs
- Accounts + Input Vault bind UX framed
- Simple prototype links for primary flows
- Report file URL + page list back to orchestrator
